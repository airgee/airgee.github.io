git的简单使用

2019-09-19 1 min read # git

git工作区

git仓库、暂存区、工作区

git参数配置

1、设置用户名
git config —global user.name ‘username’
2、设置用户名邮箱
git config —global user.email ‘email’
3、查看设置
config 配置有system级别 global（用户级别） 和local（当前仓库）三个 设置先从system-》global-》local 底层配置会覆盖顶层配置 分别使用--system/global/local 可以定位到配置文件
查看系统config
git config --global --list
查看当前用户（global）配置
git config --global --list
查看当前仓库配置信息
git config --local --list
查看综合config
git config --list

常用命令

  • 初始化:
    git init
  • 查看文件存储区域:
    git status
  • 将某个文件更新从工作区提交到暂存区:
    git add 文件路径
  • 提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)
    git add -u
  • 提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件
    git add .
  • 提交所有变化:
    git add -A
  • 拉取远程仓库文件:
    git pull
  • 将暂存区的文件更新提交到git仓库:
    git commit -m'说明'
  • 删除文件:
    git rm 文件路径
  • 本地连接远程仓库：
    git remote add origin git库地址
  • 本地初始化并连接远程仓库后，提交数据：
    git push -u origin master
  • 克隆远程仓库到本地:
    git clone 地址
  • 提交到远程仓库:
    git push

常用操作

为不同项目设置不同的用户名和邮箱

有时候我们可能会使用多个git账号，给不同的项目设置不同的个人信息很常见。只需要进入到项目中，项目完成初始化之后，运行以下命令:

git  config  user.name "用户名"
git  config  user.email "邮箱"

也可以通过更改全局配置的方式，但那样不够灵活，除非是某个账号不用了，以后只用另外一个账号

常见问题

  1. fatal: The remote end hung up unexpectedly

将数据发送到远程系统时，智能HTTP传输使用的缓冲区的最大大小（以字节为单位）。
对于大于此缓冲区大小的请求，HTTP / 1.1 Transfer-Encoding: chunked用于避免在本地创建大量包文件。默认值为1 MiB，足以满足大多数请求。
The remote end hung up unexpectedly while git cloning

具体看是哪种传输方式(ssh还是http)，运行以下命令之一进行配置即可

git config --global http.postBuffer 524288000
git config --global ssh.postBuffer 524288000
